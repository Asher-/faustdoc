Using and Developing Architectures Files

Introduction

A Faust program describes a signal processor, a pure DSP computation that maps input signals to output signals. It says nothing about audio drivers or controllers (like GUI, OSC, MIDI, sensors) that are going to control the DSP. This additional information is provided by architecture files. We assume here a general understanding of the architecture files model. In this tutorial, we'll explain how the machinery works:

  • starting with a simple example
  • gradually enriching it to show how multiple controllers can be used (OSC, http, MIDI etc.)
  • showing how the architectures files can be used with statically generated C++ files, or dynamically generated dsp using the LLVM or Interpreter backends
  • finally demonstrating how to write simple architecture files or combine, extend and use existing ones.
